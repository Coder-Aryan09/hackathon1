Lecture 1 : Introduction to Development

  • Basic Networking: Computers share data by sending bits (0s and 1s) as electrical or light signals. Small networks can use direct cables, but larger ones need switches to connect devices in a LAN. To connect different LANs, routers are used. The Internet is a massive network of networks connected by routers.
  • IP Addresses: Every device on the Internet has a unique IP. IPv4 uses 32 bits (~4.3 billion addresses), but IPv6 uses 128 bits (~3.4√ó1038 addresses). Private IPs (like 192.168.x.x) work inside local networks, while public IPs identify networks globally. Routers forward packets hop by hop using these addresses.
  • Packets and TCP/IP: Large data is split into packets containing a data piece, source/destination IPs, and sequence numbers. Packets may take different routes, and the receiver reassembles them. TCP ensures reliable, ordered delivery and retransmits missing packets.
  • MAC Addresses: Each device has a unique 48-bit MAC address (e.g., 3C:22:FB:A3:B4:C5) hard-coded into hardware. Switches use MAC addresses to deliver data locally, while routers use IP addresses for traffic across networks.
  • Port Numbers: Ports identify services on a device:
    • Well-known ports (0‚Äì1023): Reserved for common services like HTTP (80), HTTPS (443), SSH (22), DNS (53).
    • Registered ports (1024‚Äì49151): Used by specific apps/services like MySQL (3306), MongoDB (27017).
    • Dynamic/Ephemeral ports (49152‚Äì65535): Assigned temporarily for client-side connections.
  • DNS (Domain Name System): Converts domain names into IP addresses through:
    1. Checking local cache (browser/OS/router).
    2. Querying a recursive resolver (ISP or public like 8.8.8.8).
    3. Resolver contacts root ‚Üí TLD (.com) ‚Üí domain‚Äôs authoritative server.
    4. Authoritative server gives final IP; resolver caches it and returns it.
    Finally, the browser connects to that IP (usually on port 443 for HTTPS).


Lecture 2 : Introduction to HTML

  • HTML stands for HyperText Markup Language, the standard language used to create web pages and web applications.
  • It defines the structure of a webpage using a system of elements and tags.
  • HTML elements act as the building blocks of web pages and organize content in a meaningful way.
  • Commonly used tags include: <html>, <head>, <body>, <h1> to <h6>, <p>, <a>, <img>, etc.
  • Basic elements:
    • Headings: <h1> to <h6>
    • Paragraphs: <p>
    • Links: <a>
    • Images: <img>
  • The structure of an HTML document typically includes:
    • <!DOCTYPE html> declaration
    • <html> element
    • <head> element
    • <body> element
  • Attributes add extra information to elements, e.g., href in links (<a>) and src in images (<img>).
  • Elements can be nested within each other to build complex page layouts.
  • Semantic elements like <header>, <footer>, <article>, and <section> add meaning and improve accessibility.


Lecture 3 : HTML Lists & Tables

  • HTML supports three main types of lists: <ol> (ordered lists), <ul> (unordered lists), and <dl> (definition lists).
  • Ordered lists are used when items follow a specific sequence, while unordered lists are for items without a particular order.
  • List items are defined using the <li> tag.
  • Definition lists consist of <dt> (term) and <dd> (definition).
  • Lists can be nested inside other lists to create sublists.

  • Example of an unordered list:

    • Apple
    • Banana
    • Cherry

    Example of an ordered list:

    1. First Step
    2. Second Step
    3. Third Step
  • HTML tables are used to display data in a structured tabular format.
  • The <table> element defines a table, <tr> defines a row, <td> defines a cell, and <th> defines a header cell.
  • Tables can have captions using the <caption> tag.
  • Attributes like colspan and rowspan allow cells to span multiple columns or rows.

  • Example of a simple table:

    Student Information
    Name Age Location
    Aryan 20 City Noida
    State Uttar Pradesh
    Rohit 22 Noida, Uttar Pradesh


Lecture 4 : File Paths & Naming Conventions

Websites have many files (HTML, images, etc.). File paths tell the browser where to find each file. Correct paths ensure links and images work on any computer or server.

Relative Paths

  • Same folder: Use the filename directly. Example: <a href="about.html">About</a>.
  • Subfolder: Include the folder name. Example: <img src="images/logo.png" alt="Logo"> (if logo.png is in an "images" folder).
  • Parent folder: Use ../ to go up one level. Example: In pages/contact.html, use <img src="../images/logo.png">.

Absolute Paths

Absolute paths use the full URL and start with http:// or https://. Use them for external resources.

  • External link: <a href="https://www.example.com">Example</a>.
  • External image: <img src="https://example.com/image.png" alt="Image">.
  • Never use OS file paths (like C:\Users\) in HTML; browsers cannot access your computer‚Äôs files for security.

Rules of Thumb

  • Internal files: Use relative paths (e.g. page.html or folder/file.png).
  • External resources: Use absolute URLs (starting with https://).
  • Windows vs. macOS/Linux: Windows paths start with a drive letter and use backslashes (e.g. C:\Users\John\file.txt). macOS/Linux paths start from the root / and use forward slashes (e.g. /Users/john/file.txt).

    • <input type="checkbox"> ‚Üí checkboxes

  • Other form elements include:
    • <textarea> ‚Üí for multi-line text input
    • <select> ‚Üí for dropdown menus
  • Forms are submitted to a server using:
    • action ‚Üí specifies the URL to send the data
    • method ‚Üí specifies the HTTP method (GET or POST)
  • Form validation can be done using HTML5 attributes like required, pattern, and maxlength.
